# 06 数据字段与数据集

> WorldQuant BRAIN 提供 85,000+ 数据字段（顾问权限），普通用户也有数千个字段可用。理解数据是构建高质量 Alpha 的基础。

---

## 一、数据的基本概念

### 数据字段（Data Field）

数据字段是具有固定**类型**和**业务含义**的命名数据集合。

- 每个字段对应一个二维矩阵（行=日期，列=股票）
- 同一字段在不同时间/股票下的含义保持一致
- 例：`close`（收盘价）每天每只股票都有唯一值

### 数据集（Dataset）

数据集是相关数据字段的集合，通常来自同一数据提供商或类型。

- 例：`price_volume`（价量数据集）包含 `close`、`open`、`volume` 等字段
- 每个数据集有唯一的 `dataset_id`（短字母编号）
- 顾问可以在 Data Explorer 中按数据集浏览和筛选

### 矩阵字段（Matrix Field）

普通字段：每个日期×股票只有一个值。不需要特殊语法，直接使用字段名即可。

```
close              # 直接使用
ts_delta(close, 5) # 参与计算
```

### 向量字段（Vector Field）

特殊字段：每个日期×股票有**多个值**（如分析师预测有多位分析师、期权数据有多个行权价）。

使用前必须通过**向量运算符**转换为矩阵：
```
vec_sum(analyst_price_target)   # 将所有分析师预测求和
vec_avg(analyst_price_target)   # 求平均
```

## 二、数据类型分类

### 2.1 价量数据（Price / Volume）

最基础的数据，日频更新，适合技术面 Alpha。

| 字段 | 类型 | 含义 |
|------|------|------|
| `close` | 价格 | 收盘价 |
| `open` | 价格 | 开盘价 |
| `high` | 价格 | 最高价 |
| `low` | 价格 | 最低价 |
| `vwap` | 价格 | 成交量加权平均价格 |
| `returns` | 收益率 | 日收益率 = (close-prev_close)/prev_close |
| `volume` | 成交量 | 成交股数 |
| `adv5` | 金额 | 5 日平均成交额（Amount） |
| `adv20` | 金额 | 20 日平均成交额 |
| `cap` | 市值 | 总市值（Market Cap） |

### 2.2 基本面数据（Fundamental）

来自公司财务报表，季度/年度更新，适合价值类 Alpha。

| 字段 | 含义 |
|------|------|
| `assets` | 总资产 |
| `liabilities` | 总负债 |
| `sales` | 营业收入 |
| `earnings` | 净利润 |
| `dividends` | 股息 |
| `inventory` | 存货 |
| `debt` | 有息负债 |
| `cash` | 现金及等价物 |
| `equity` | 股东权益 |
| `operating_income` | 营业利润 |
| `ebitda` | 息税折旧摊销前利润 |
| `capex` | 资本支出 |
| `inventory_turnover` | 存货周转率（内置比率） |

### 2.3 分析师预测数据（Analyst / Estimates）

来自证券分析师研报，覆盖率较低但信号价值高。

| 字段前缀 | 含义 |
|----------|------|
| `est_eps` | 分析师预测每股收益（EPS） |
| `etz_eps` | 修订版 EPS 预测 |
| `est_revenue` | 分析师预测营收 |
| `fn_*` | Financial Notes 数据集字段 |
| `analyst_*` | 分析师评级/目标价（向量字段） |

### 2.4 替代数据（Alternative Data）

非传统金融数据，通常为高频或独特视角，竞争较低。

| 字段前缀 | 数据集 | 说明 |
|----------|--------|------|
| `scl12_buzz` | 情绪量 | 股票被讨论的相对声量 |
| `scl12_*` | 情绪数据集 | 情绪分析系列字段 |
| `wq_*` | WorldQuant 内部 | 专有数据字段 |

### 2.5 分组/分类字段（Grouping Fields）

这些字段本身不直接做信号，而是用于 `group_*` 运算符的分组维度。

| 字段 | 含义 |
|------|------|
| `sector` | GICS 一级行业（11 个板块） |
| `industry` | GICS 二级行业 |
| `subindustry` | GICS 三级子行业 |
| `market` | 交易所（NYSE, NASDAQ 等） |
| `country` | 国家 |
| `exchange` | 具体交易所代码 |
| `currency` | 交易货币 |

## 三、Data Explorer 使用指南

### 访问方式

BRAIN 平台 → **Data** 菜单 → Data Explorer（NLP 驱动搜索）

### 搜索技巧

**3S 原则**：Short（简短）、Simple（简单）、Straightforward（直接）

```
✅ 好的搜索词：
"earnings per share"  或  "eps"（同时搜全称和缩写）
"insider trading"  或  "trade by people inside company"
"implied volatility"  或  "IV"

❌ 避免的搜索方式：
过长的描述性语句（NLP 可能偏离）
```

### 搜索维度

- 按**关键词**搜索字段或数据集
- 按**覆盖率**过滤（低覆盖字段需要 backfill 处理）
- 按**字段类型**过滤（Matrix / Vector）
- 按**Alpha 使用数量**排序（少人用 = 更独特信号）
- 按**用户数量**过滤

### Data Explorer 筛选参数

搜索时需要预先设定：
- **Region**（地区）
- **Universe**（股票池）
- **Delay**（延迟）

某些字段只在特定 Region/Delay 下有数据。

## 四、快速了解新数据字段

拿到一个不熟悉的字段 `datafield`，用以下表达式探索其特性：

| 序号 | 表达式 | 获得的信息 |
|------|--------|-----------|
| 1 | `datafield` | 覆盖率（Long+Short Count / Universe Size） |
| 2 | `datafield != 0 ? 1 : 0` | 非零覆盖（每日有多少股票有非零值） |
| 3 | `ts_std_dev(datafield, N) != 0 ? 1 : 0` | 数据更新频率（N=5/22/66 对应周/月/季） |
| 4 | `abs(datafield) > X` | 数据的范围和边界（调整 X 查看分布） |
| 5 | `ts_rank(datafield, 1000) > X` | 数据分布（中位数附近 X≈0.5） |

**模拟设置**：使用 `Neutralization=None, Decay=0` 以获得最原始的数据探索结果。

**实例**：
```
# 探索 earnings 字段
earnings               → 覆盖率：Long+Short ≈ 2000（约66%的TOP3000有数据）
earnings != 0 ? 1 : 0  → 非零覆盖（季报发布时才变化）
ts_std_dev(earnings, 22) != 0 ? 1 : 0  → N=66时覆盖率高 → 季度更新数据
abs(earnings) > 1e8    → 1亿以上利润的公司数量
```

## 五、数据覆盖率处理

低覆盖率数据字段需要手动处理 NaN。

### 策略一：时间序列回溯填充

```
ts_backfill(earnings, 60)    # 向前回溯 60 天，用最近有效值填充
```

### 策略二：行业均值填充

```
group_backfill(earnings, sector, 5)  # 用同行业均值填充
```

### 策略三：备选字段替换

```
if_else(is_nan(etz_eps), ts_backfill(est_eps, 60), etz_eps)
```

### 策略四：过滤低覆盖股票

```
rank(if_else(is_nan(earnings), NaN, -earnings))
# 直接忽略没有盈利数据的股票（rank 会跳过 NaN）
```

## 七、数据使用注意事项

### ⚠️ 财务数据时间对齐

财务报告有**发布延迟**：季报通常在季度结束后 30-60 天发布。  
使用财务数据时务必确认：
- 数据是基于**已公布数据**（避免前视偏差）
- 通常平台已处理，但需要注意 `Delay=1` 设置

### ⚠️ 向量字段必须转换

```
# 错误：直接使用向量字段会报错
rank(analyst_price_target)   ❌

# 正确：先用向量运算符合并
rank(vec_avg(analyst_price_target))  ✅
```

### ⚠️ 数据单位

不同字段单位可能不兼容：
- `close` 单位：价格
- `cap` 单位：价格 × 股数
- `volume` 单位：股数

启用 `Unit Handling=Verify` 可以检测单位不兼容，避免无意义的运算。
