# 04 模拟设置详解

> 模拟设置决定了 Alpha 回测的市场范围、交易假设和数据处理规则，是每次 Simulate 的基础配置。

## 一、设置面板位置

在 BRAIN 平台 Simulate 页面，点击右上角 **Settings** 按钮打开设置面板，修改后点击 **Apply** 生效。

---

## 二、各项设置详解

### 2.1 Language（语言）

目前 BRAIN 平台支持 **Fast Expression**（FE）语言。

---

### 2.2 Instrument Type（金融工具类型）

当前仅支持 **Equity（股票）**。

---

### 2.3 Region（地区）

| 地区 | 可用性 |
|------|--------|
| **USA（美国）** | 所有用户可用 |
| **Europe（欧洲）** | 仅研究顾问（Consultant）可用 |
| **Asia（亚洲）** | 仅研究顾问可用 |
| **GLB（全球）** | 仅研究顾问可用 |

---

### 2.4 Universe（股票池）

Universe 是 BRAIN 平台按流动性筛选出的交易标的集合，以**平均日成交额**排名。

| Universe | 含义 |
|----------|------|
| `TOP3000` | 美国最具流动性的 3000 只股票（默认） |
| `TOP2000` | 前 2000 只 |
| `TOP1000` | 前 1000 只 |
| `TOP500` | 前 500 只 |
| `TOP200` | 前 200 只（流动性最高，竞争最激烈） |

**选择建议**：
- 较小的 Universe（TOP500）得分系数更高，但信号覆盖范围更窄
- 较大的 Universe（TOP3000）信号多但得分系数较低
- 提交时需通过 **Sub-Universe Test**（详见第 5 篇）

---

### 2.5 Delay（延迟）

Delay 表示"决策时间与实际交易时间的滞后"，核心是**防止前视偏差（Look-ahead Bias）**。

| Delay 值 | 含义 | 推荐程度 |
|----------|------|----------|
| **Delay 1**（推荐） | 用 T 日数据，T+1 日交易 | ★★★★★ 更真实、标准 |
| **Delay 0** | 用 T 日数据，T 日收盘前交易 | ⚠️ 较激进，需更高 Sharpe/Fitness |

```
举例：
  Delay 1：
    今天收盘后看到信号 → 明天开盘后交易
    更符合实际操作，避免当日滑点问题

  Delay 0：
    今天盘中/收盘前看到信号 → 今天收盘前交易
    适合日内策略，但要求更高 IS 表现
```

**提交标准差异**：
- Delay 1：Sharpe ≥ 1.25，Fitness ≥ 1.0
- Delay 0：Sharpe ≥ 2.0，Fitness ≥ 1.3

---

### 2.6 Decay（衰减）

Decay 对 Alpha 权重进行线性加权平滑，降低因每日信号波动产生的**高换手率**。

**公式**：

```
Decay_linear(x, n) = (x[t]*n + x[t-1]*(n-1) + ... + x[t-n+1]*1) / (n + (n-1) + ... + 1)
```

| Decay 值 | 效果 |
|----------|------|
| `0` | 无衰减，使用当日原始信号 |
| `4`（默认） | 近 4 天线性加权平均 |
| `10+` | 强平滑，换手率极低，但信号稀释严重 |

**注意事项**：
- Decay 只接受**非负整数**，负值或非整数会导致模拟失败
- 过大的 Decay 会显著衰减 Alpha 信号，降低 Fitness
- 通常 Decay = 4~6 是较好的起点

---

### 2.7 Truncation（截断）

限制**单只股票的最大持仓权重**，防止持仓过于集中。

| 值 | 含义 |
|----|------|
| `0` | 不截断（无限制） |
| `0.01` | 默认，单股最大权重 1% |
| `0.05~0.1` | 推荐范围（5%~10%） |

**注意**：提交标准要求单股最大权重 < 10%，Truncation 是确保通过 **CONCENTRATED_WEIGHT** 检查的有效手段。

```
# 如果你的 Alpha 集中于少数股票（如行业龙头），设置 Truncation = 0.05
# 可以有效分散风险并通过权重检查
```

---

### 2.8 Neutralization（中性化）

中性化操作使 Alpha 向量均值为零，实现市场/行业中性。

| 选项 | 操作 | 说明 |
|------|------|------|
| `Market`（默认） | `alpha -= mean(alpha)` | 全市场中性：做多做空敞口相等 |
| `Industry` | 对每个行业分别做市场中性 | 行业中性：同行业内多空平衡 |
| `Subindustry` | 对每个子行业分别做市场中性 | 子行业中性：更细粒度 |
| `None` | 不做中性化 | 适合特殊策略，但风险更高 |

```
# 示例：Market Neutralization 后的效果
原始 alpha = [0.5, 0.2, 0.3, -0.1, 0.4]
mean = 0.26
中性化后 = [0.24, -0.06, 0.04, -0.36, 0.14]
→ 多头 ≈ 空头，净敞口为零
```

**选择建议**：
- 大多数 Alpha 使用 `Market` 中性化（默认）
- 有行业倾斜风险时使用 `Industry`
- `None` 适用于有意识地持有方向性头寸（较少见）

---

### 2.9 Pasteurize（数据净化）

控制是否将 Universe 之外的股票数据设为 NaN。

| 选项 | 行为 |
|------|------|
| `On`（默认） | Universe 外的股票输入数据变为 NaN |
| `Off` | 使用全部可用股票数据 |

**使用场景**：
- `Off` + 手动 `pasteurize(x)` 算子：可以精细控制哪些数据参与截面计算

```
# Off 模式下手动净化示例（用于对比 Universe 内外的信号差异）
group_rank(pasteurize(sales_growth), sector) - group_rank(sales_growth, sector)
```

---

### 2.10 NaN Handling（NaN 处理）

控制如何处理数据中的缺失值（NaN）。

| 选项 | 行为 |
|------|------|
| `Off`（默认） | 保留 NaN，运算结果为 NaN |
| `On` | 自动填充 NaN（时序算子：NaN→0；组算子：NaN→组均值） |

⚠️ **`On` 的风险**：  
`NaNHandling=On` 提高了数据覆盖率，但可能引入模糊信息。  
例如：`ts_zscore(x, d) = 0` 可能是"数据为 0"或"所有天均为 NaN"，两种含义完全不同。

**推荐做法**（`Off` 模式 + 手动处理）：
```
# 用备选字段替换 NaN
if_else(is_nan(ts_zscore(etz_eps, 252)), ts_zscore(est_eps, 252), ts_zscore(etz_eps, 252))

# 用时间序列回溯填充
ts_backfill(earnings, 60)

# 用行业均值填充
group_backfill(earnings, sector, 5)
```

---

### 2.11 Unit Handling（单位检查）

当启用时，如果表达式中出现单位不兼容的运算（如"价格 + 成交量"），会弹出警告。

| 选项 | 行为 |
|------|------|
| `Verify`（默认） | 检查单位相容性，不兼容时显示警告 |
| `Off` | 不检查，静默运行 |

## 三、完整 API 设置结构（模拟提交用）

通过 API 提交模拟时，需在请求体中提供完整的设置对象：

```json
{
  "type": "REGULAR",
  "settings": {
    "instrumentType": "EQUITY",
    "region": "USA",
    "universe": "TOP3000",
    "delay": 1,
    "decay": 4,
    "neutralization": "MARKET",
    "truncation": 0.01,
    "pasteurize": "ON",
    "nanHandling": "OFF",
    "unitHandling": "VERIFY",
    "language": "FASTEXPR",
    "visualization": false
  },
  "regular": "rank(-ts_delta(close, 5))"
}
```

**参数枚举值**：

| 参数 | 可选值 |
|------|--------|
| `region` | `"USA"`, `"EUR"`, `"ASI"`, `"GLB"` |
| `universe` | `"TOP3000"`, `"TOP2000"`, `"TOP1000"`, `"TOP500"`, `"TOP200"` |
| `delay` | `0`, `1` |
| `neutralization` | `"MARKET"`, `"INDUSTRY"`, `"SUBINDUSTRY"`, `"NONE"` |
| `pasteurize` | `"ON"`, `"OFF"` |
| `nanHandling` | `"ON"`, `"OFF"` |
| `unitHandling` | `"VERIFY"`, `"OFF"` |

## 五、测试周期（IS/OS）

| 周期 | 时间范围 | 说明 |
|------|---------|------|
| **In-Sample（IS）** | 约 7 年前 ~ 2 年前 | 模拟训练期，可以看到结果 |
| **Semi Out-of-Sample** | 最近 2 年 | 隐藏，提交后才部分可见 |
| **Out-of-Sample（OS）** | 提交后实时积累 | 决定 Alpha 是否 ACTIVE |

模拟时只能看到 IS 期间的表现。提交后系统会在 OS 期间持续监测，如果 OS 表现持续差则 Alpha 会被 DECOMMISSIONED。
